# 任务拆解

把大方案拆成小任务，让实现过程可执行、可验证、可追踪。

## 目标

输出一份带有依赖关系、验证方式和完成标准的任务清单。

## 拆解步骤

1. 识别最小可交付闭环。
2. 划分任务边界，避免一个任务覆盖过多问题。
3. 标注每个任务的目标和涉及模块。
4. 标注前置依赖和建议顺序。
5. 为每个任务补充验证方式。

## 输出

输出至少包含以下内容：

- 任务编号
- 任务目标
- 涉及文件或模块
- 前置依赖
- 实施说明
- 验证方式
- 风险提示

## 工作约束

- 不要输出只有标题没有完成标准的任务。
- 不要把多个独立问题粗暴捆绑在一起。
- 不要把任务拆到失去执行效率。
- 不要遗漏关键依赖关系。
