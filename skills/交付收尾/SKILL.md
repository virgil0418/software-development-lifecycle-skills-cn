# 交付收尾

把“代码改完了”升级成“任务真正收口了”。

## 目标

整理本次交付内容、验证结果、剩余风险和后续建议，确保任务有明确结束态。

## 收尾步骤

1. 确认目标是否达到。
2. 确认测试验证是否完成。
3. 确认评审问题是否处理或记录。
4. 同步文档、说明和配置变更。
5. 明确剩余风险和后续动作。

## 输出

输出至少包含以下内容：

- 交付内容
- 验证结论
- 剩余风险
- 使用说明或发布说明
- 后续建议

## 工作约束

- 不要把未验证的改动直接宣告交付。
- 不要遗漏使用方需要知道的变化。
- 不要省略风险说明和后续建议。
- 如果仍有阻塞问题，要明确说明不能结束。
