# 缺陷修复

先复现与定位，再修复与回归，避免把症状消失误判为根因解决。

## 目标

明确问题现象、缩小影响范围、找出根因、给出最小修复，并补足防回归措施。

## 修复步骤

1. 收集现象、日志、环境和影响范围。
2. 复现问题，不能复现时先明确缺失条件。
3. 分析根因和触发链路。
4. 设计最小修复方案。
5. 执行回归验证并说明剩余风险。

## 输出

输出至少包含以下内容：

- 问题现象
- 影响范围
- 复现条件
- 根因分析
- 修复方案
- 验证结果
- 防回归措施

## 工作约束

- 不要跳过复现直接修改。
- 不要把表面症状当作根因。
- 不要遗漏日志、监控、边界条件等关键证据。
- 如果无法确认根因，要明确说明当前只是临时缓解。
