# 场景：lessons 示例

## 场景描述

在一个移动端远程控制项目中，团队一开始打算按照通用 SSE 规范实现完整解析器，后来通过直接实测服务端事件流，发现当前服务端只返回单行 `data:` JSON。团队因此避免了过度设计，并希望把这个经验沉淀下来。

## 推荐 skill 流程

1. 缺陷修复 或 编码实施
2. 代码评审
3. lessons

## 应得到的最小产物

- 一条包含场景、反模式、正确做法和适用边界的经验记录

## 示例内容

### 场景

调研外部 SSE 接口时，需要确认服务端到底返回什么格式。

### 反模式或问题

先按完整 SSE 规范实现复杂解析器，再通过客户端运行日志去反推真实返回格式。

### 正确做法

先直接使用 curl 或等价工具连接真实接口，确认事件流格式，再决定解析器复杂度。

### Lesson

面对外部 API 或协议时，优先直接验证真实行为，而不是先按想象实现。

### 适用边界

适用于依赖外部接口、SSE、Webhook、第三方 SDK 或平台行为的场景；如果接口行为已稳定且有强契约文档，可适当减少实测成本。
