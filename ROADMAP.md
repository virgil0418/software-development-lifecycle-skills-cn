# 路线图

## V0.1 已完成

- 建立中文 skill 仓库目录
- 定义生命周期主线技能
- 增加模板、示例、脚本
- 建立多工具安装入口

## V0.2 已完成

- 强化 `需求澄清`、`技术方案`、`任务拆解` 的文档决策能力
- 增加 `WORKING` 与 `lessons` 两个横向 skill
- 在主控与若干执行 skill 中加入文档触发意识
- 让仓库从阶段流程扩展为文档—过程—经验闭环

## V0.3 已完成

- 新增 `技能编写` skill（元能力：用 TDD 方法编写 skill）
- 强化 `测试先行`：增加铁律、Red-Green-Refactor 分阶段步骤、合理化反驳表、红旗清单
- 强化 `系统化调试`：增加铁律、四阶段结构、合理化反驳表、红旗清单、3 次失败升级机制
- 增强 `任务拆解`：文件结构前置、原子粒度步骤、TDD 步骤嵌入、精确输出格式（含代码和验证命令）
- 同步更新任务清单模板
- 更新能力地图，补充元能力层

## V0.4 已完成

- 新增 `完成验证` 纪律 skill（全局门控：没有验证证据就不许宣称完成）
- 拆分 `代码评审` 为 `代码评审`（发起评审）+ `接收评审`（接收反馈）两个独立 skill
- 强化 `编码实施`：增加前置检查、5 步执行节奏、阻塞处理表、质量 Gate
- 在总控和关键 skill 中增加 `## 强制依赖` 段落，建立 REQUIRED 级别依赖链
- 契约测试从 11 条扩展到 18 条，覆盖新增 skill 和强制依赖
- 更新能力地图，补充完成验证和接收评审

## V0.5 计划

## V0.6 计划
